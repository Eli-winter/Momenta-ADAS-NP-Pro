## v37.12.0 — 修复 systemd 重启残留僵尸进程导致端口占用 `2026-05-21`

## v37.11.0 — 后端主动同步构建状态，不再依赖前端轮询 `2026-05-21`

- 新增后台线程 `build-sync`，每 10 秒主动扫描所有 context 下 `status=构建中` 且有 `pipeline_url` 但无 `final_version` 的构建，调用 pipeline API 写入结果
- 根因：v33.1.0 修复切换车型写错数据问题时，引入了"切换车型强制停止轮询"逻辑，导致用户中途切走后构建完成永远检测不到；后端主动扫描彻底绕开这个隐患，与前端是否在线无关

## v36.1.0 — 修复拟合曲线散点图空白区坐标显示 `2026-05-20`

- 光标在数据点密集区之间的空白处时，坐标提示（速度 / TTC）现在正常显示
- 根因：Plotly 默认 `hoverdistance: -1`（无距离限制），光标在图内任意位置都会 snap 到最近点并触发 `plotly_hover`，导致自定义 tooltip 常驻、坐标 label 被压制；加 `Plotly.relayout(pd, {hoverdistance: 12})` 后，距点 >12px 的空白区触发 `plotly_unhover`，坐标 label 即时显示
- tooltip 内去除多余的「ESS 链接」纯文本（原为剥离 `<a>` 标签后的残留），仅保留底部可点击链接

## v36.0.0 — 修复拟合曲线散点图交互功能 `2026-05-20`

- 恢复三个重叠率（100% / 50% / -50%）散点图的全部交互：空白区显示光标坐标、数据点 hover 显示速度 + TTC + event 名称 + ESS 链接、ESS 链接可点击
- 根因：前一版本注入脚本使用 `window.addEventListener("load")` 并在未找到 `.nsewdrag` 时直接 `return`，lazy iframe 场景下 Plotly 尚未完成渲染即退出，导致 mousemove 监听器从未挂载；同时 `data.event` 无空值保护，tooltip 定位失败
- 改为 `setTimeout(init, 300)` + 轮询 `pd._fullLayout`；新增 `setupDrag()` 携带 200ms 重试逻辑，确保 `.nsewdrag` 就绪后再绑定
- 同步更新本地及服务器 `fit_polyline.py`，后续重新标定生成的图表同样生效
